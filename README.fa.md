## معرفی پروژه

هایر‌فلو یک پلتفرم مدیریت استخدام چندمستأجری (Multi-tenant) است که به تیم‌های منابع انسانی کمک می‌کند تا:

- موقعیت‌های شغلی را منتشر و مدیریت کنند.
- کاندیداها را در یک پایگاه داده مرکزی نگه‌داری کنند.
- درخواست‌های شغلی را در یک **پایپ‌لاین** (Kanban) جابه‌جا کنند.
- مصاحبه‌ها را برنامه‌ریزی و **فرم‌های ارزیابی (Scorecard)** ثبت کنند.
- پیشنهادهای شغلی را با گردش‌کار تأیید چندمرحله‌ای ارسال کنند.
- با کاندیداها از طریق ایمیل ارتباط برقرار کنند.
- یک **صفحه عمومی فرصت‌های شغلی** برای ثبت درخواست بدون نیاز به حساب کاربری داشته باشند.

## پشته فناوری

| لایه | فناوری |
|---|---|
| Frontend | React 18 + TypeScript + Vite |
| استایل | Tailwind CSS + shadcn/ui |
| مدیریت State | TanStack Query + Zustand |
| فرم‌ها | React Hook Form + Zod |
| Backend | Lovable Cloud (مبتنی بر Supabase) |
| پایگاه داده | PostgreSQL با Row Level Security |
| Edge Functions | Deno |
| ذخیره فایل | Supabase Storage |
| ایمیل | Resend |
| امضای الکترونیکی | HelloSign / Dropbox Sign |
| فونت | **Vazirmatn** |

## راه‌اندازی محلی

```bash
# نصب وابستگی‌ها
bun install

# اجرای سرور توسعه
```

## نقش‌های کاربری

این سامانه سه نقش اصلی دارد:

| نقش | شرح | دسترسی‌ها |
|---|---|---|
| **مدیر کل** (`site_admin`) | مدیر سازمان | همه چیز: مدیریت کاربران، همه شغل‌ها، تنظیمات سازمان، برندینگ |
| **مدیر شغل** (`job_admin`) | مسئول یک یا چند موقعیت | شغل‌های اختصاص‌یافته، پیشنهادها، تأیید گردش‌کار |
| **همکار** (`basic`) | عضو معمولی تیم | شغل‌ها و کاندیداهای اختصاص‌یافته، ثبت بازخورد |

نقش‌ها در جدول مستقل `user_roles` نگه‌داری می‌شوند (طبق بهترین روش‌های امنیتی)، نه در جدول پروفایل.

## راهنمای استفاده از هر بخش

### داشبورد
- نمایش متریک‌های کلیدی: موقعیت‌های فعال، تعداد کل متقاضیان، تعداد در حال مصاحبه، کاندیداهای نهایی‌شده.
- مصاحبه بعدی و آخرین فعالیت‌های تیم.
- بسته به نقش کاربر، نمای متفاوتی نمایش داده می‌شود.

### موقعیت‌های شغلی
- ایجاد آگهی شغل با فیلدهای کامل: عنوان، توضیحات، نوع همکاری، محدوده حقوق، مکان.
- **سؤالات سفارشی**: می‌توانید برای هر موقعیت، سؤالات اختصاصی متقاضیان تعریف کنید.
- **مدیریت تیم**: تعیین اعضای تیم با دسترسی‌های مختلف.
- اشتراک‌گذاری آگهی از طریق ایمیل.
- وضعیت‌های شغل: پیش‌نویس، در انتظار تأیید، باز، بسته.

### کاندیداها
- بانک اطلاعاتی کاندیداها مستقل از شغل.
- آپلود و پارس خودکار رزومه (با AI).
- امتیازدهی، یادداشت‌ها و کامنت تیمی.
- ثبت درخواست برای یک یا چند شغل.

### درخواست‌ها (Applications)
- اتصال کاندیدا به شغل.
- مدیریت کاور لتر، یادداشت‌های ارزیابی، تاریخچه کامل.
- عملیات: استخدام، رد، پس گرفتن.

### پایپ‌لاین (Kanban)
- مشاهده درخواست‌ها در مراحل: ثبت‌شده، تماس تلفنی، مصاحبه فنی، مصاحبه نهایی، پیشنهاد، استخدام، رد.
- جابه‌جایی با Drag & Drop.
- عملیات گروهی (رد دسته‌جمعی).

### مصاحبه‌ها
- برنامه‌ریزی مصاحبه با تعیین زمان، مکان یا لینک جلسه، هیئت مصاحبه.
- ارسال خودکار دعوت‌نامه ایمیلی و فایل ICS برای تقویم.
- ثبت **فرم ارزیابی (Scorecard)** برای هر مصاحبه‌کننده با توصیه نهایی.

### پیشنهادها
- ساخت پیشنهاد شغلی با جزئیات حقوق، پاداش، سهام، تاریخ شروع.
- **گردش‌کار تأیید چندمرحله‌ای**: افزودن چند تأییدکننده و تغییر خودکار وضعیت پیشنهاد.
- تولید PDF حرفه‌ای پیشنهاد.
- ارسال برای امضای الکترونیکی از طریق HelloSign.
- پاسخ کاندیدا: پذیرفته / رد شد → به‌روزرسانی خودکار وضعیت درخواست.

### وظایف (Tasks)
- ایجاد وظایف داخلی و تخصیص به اعضای تیم.
- موعد، اولویت و وضعیت.

### قالب‌ها (Templates)
- ساخت قالب‌های ایمیل قابل استفاده مجدد.

### صفحه عمومی فرصت‌های شغلی (`/careers`)
- یک سایت عمومی برای نمایش موقعیت‌های باز.
- متقاضیان بدون نیاز به حساب می‌توانند درخواست ثبت کنند.
- پاسخ به سؤالات سفارشی و آپلود رزومه.

### تنظیمات
- **پروفایل سازمان**: نام، آدرس، اطلاعات تماس.
- **برندینگ**: نام پلتفرم، لوگو، رنگ‌های اصلی و فرعی (در سایدبار و ایمیل‌ها اعمال می‌شود).
- **اعلان‌ها**: تنظیم اعلان‌های ایمیلی.
- **مدیریت کاربران**: دعوت، ویرایش نقش، حذف کاربران.
- **حساب کاربری**: ویرایش اطلاعات، تغییر رمز، حذف حساب.

## حساب کاربری آزمایشی (Demo)

برای آشنایی سریع با سامانه می‌توانید از حساب آزمایشی استفاده کنید:

- **ایمیل:** `demo@hireflow.app`
- **رمز عبور:** `Demo123!`

یا روی دکمه **«ورود به نسخه آزمایشی»** در صفحه ورود کلیک کنید.

> با هر ورود، داده‌های آزمایشی به‌صورت خودکار بازنشانی می‌شوند. همچنین از منوی پروفایل می‌توانید گزینه **«بازنشانی محتوای آزمایشی»** را اجرا کنید.

## یکپارچه‌سازی‌ها

### ایمیل (Resend)
- تمام ایمیل‌های تراکنشی (تأیید درخواست، دعوت مصاحبه، پیشنهاد، رد) از طریق Resend ارسال می‌شوند.
- نیاز به متغیر محرمانه: `RESEND_API_KEY`.

### امضای الکترونیکی (HelloSign)
- ارسال نامه پیشنهاد برای امضای دیجیتال کاندیدا.
- نیاز به متغیر محرمانه: `HELLOSIGN_API_KEY`.
- در صورت پیکربندی نشدن، تمام عملکردهای دیگر فعال هستند.

### پارس رزومه ( AI )
- استخراج خودکار اطلاعات از فایل رزومه (PDF/DOCX) با کمک Gemini.


## نکته درباره RTL و فونت فارسی

- چیدمان کل اپ راست‌چین (`dir="rtl"`) است.
- فونت پیش‌فرض **Vazirmatn** از Google Fonts بارگذاری می‌شود.
- پلاگین `tailwindcss-rtl` نصب شده — کلاس‌های `ml-*`، `mr-*`، `pl-*`، `pr-*` و … به‌صورت خودکار در حالت RTL آینه می‌شوند.
- توابع کمکی برای اعداد فارسی در `src/lib/fa.ts`:
  ```ts
  import { toFaDigits, formatFaDate, formatFaDateTime, formatFaNumber } from '@/lib/fa';
  
  toFaDigits(1234);            // "۱۲۳۴"
  formatFaDate(new Date());    // "۱۴ آذر ۱۴۰۴"
  formatFaDateTime(date);      // "۱۴ آذر ۱۴۰۴، ۱۴:۳۰"
  ```
- ایمیل و شناسه‌ها به‌صورت LTR نمایش داده می‌شوند (با `dir="ltr"`).
- داده‌های دیتابیس (تاریخ‌های ISO، اعداد) دست‌نخورده باقی می‌مانند؛ فقط نمایش آنها لوکالایز شده است.
